Market Segmentation in Python: A Quarto Tutorial with Synthetic Data

Survey, Transaction, and CRM Segmentation with Pyodide-Friendly Python

Author

OpenAI

Overview

This tutorial is organized around three common segmentation settings:

  1. Survey data with attitudinal and preference variables.
  2. Transaction data with customer-level behavioral features such as recency, frequency, and spending.
  3. CRM data with mixed numeric and categorical account fields.

The emphasis is on a workflow that is browser-friendly and relatively Pyodide-compatible:

  • numpy
  • pandas
  • matplotlib
  • scikit-learn

That means avoiding packages that are excellent on the desktop but less convenient in browser runtimes.

What this tutorial does

For each data type, the tutorial will:

  • generate a synthetic dataset with a known latent segment structure,
  • preprocess the data,
  • compare candidate numbers of clusters,
  • fit a segmentation model,
  • visualize the result,
  • produce segment-profiling tables.

Pyodide and Quarto notes

This file is written as a normal Quarto .qmd using executable Python chunks. It can be rendered locally with Quarto and Jupyter. It is also designed to be Pyodide-friendly in the sense that it relies on packages that are commonly available in Pyodide-based browser execution environments.

A few practical notes:

  • Keep datasets modest in size for browser execution.
  • Prefer matplotlib over heavier plotting stacks.
  • Avoid file I/O dependencies.
  • Favor scikit-learn pipelines and transformers over niche clustering packages.

def evaluate_kmeans_range(X, k_values, random_state=42, n_init=20):
    rows = []
    for k in k_values:
        model = KMeans(n_clusters=k, random_state=random_state, n_init=n_init)
        labels = model.fit_predict(X)
        rows.append({
            "k": k,
            "inertia": model.inertia_,
            "silhouette": silhouette_score(X, labels)
        })
    return pd.DataFrame(rows)


def add_segment_order(summary_df, sort_col, ascending=True):
    ordered = summary_df.sort_values(sort_col, ascending=ascending).copy()
    ordered.insert(0, "segment_rank", range(1, len(ordered) + 1))
    return ordered


def plot_metric_curve(df_metrics, x_col="k", y_cols=("inertia", "silhouette"), title_prefix=""):
    fig, axes = plt.subplots(1, 2, figsize=(12, 4))

    axes[0].plot(df_metrics[x_col], df_metrics[y_cols[0]], marker="o")
    axes[0].set_title(f"{title_prefix}Inertia by k")
    axes[0].set_xlabel("k")
    axes[0].set_ylabel("Inertia")

    axes[1].plot(df_metrics[x_col], df_metrics[y_cols[1]], marker="o")
    axes[1].set_title(f"{title_prefix}Silhouette by k")
    axes[1].set_xlabel("k")
    axes[1].set_ylabel("Silhouette")

    plt.tight_layout()
    plt.show()


def plot_pca_segments(X, labels, title="PCA view of segments"):
    if hasattr(X, "toarray"):
        X = X.toarray()
    pca = PCA(n_components=2, random_state=42)
    coords = pca.fit_transform(X)

    plt.figure(figsize=(7, 5))
    scatter = plt.scatter(coords[:, 0], coords[:, 1], c=labels)
    plt.title(title)
    plt.xlabel("PC1")
    plt.ylabel("PC2")
    plt.show()


def top_categories_by_segment(df, segment_col, cat_cols):
    pieces = []
    for col in cat_cols:
        mode_vals = (
            df.groupby(segment_col)[col]
            .agg(lambda s: s.mode().iloc[0] if not s.mode().empty else np.nan)
            .rename(col)
        )
        pieces.append(mode_vals)
    return pd.concat(pieces, axis=1)

1. Survey segmentation

Framing

Survey segmentation is common when you have attitudinal or preference variables such as:

  • price sensitivity,
  • quality emphasis,
  • environmental concern,
  • novelty seeking,
  • channel preference.

In practice, survey segmentation often uses ordinal scales or one-hot encoded categories. To keep this tutorial Pyodide-friendly, we will build a synthetic survey with Likert-like numeric items and use K-means after standardization.

2. Transaction segmentation

Framing

Transaction segmentation usually starts with customer-level feature engineering rather than raw order lines. Common features include:

  • recency,
  • purchase frequency,
  • monetary value,
  • average order value,
  • discount reliance,
  • category breadth.

This section generates synthetic RFM-style data plus two additional behavioral variables.

3. CRM segmentation with mixed data

Framing

CRM and account-level segmentation often combines:

  • numeric variables such as tenure, spend, and support load,
  • categorical variables such as region, plan type, channel, and industry.

A mixed-type workflow can be handled cleanly with a ColumnTransformer that standardizes numeric columns and one-hot encodes categorical columns, followed by clustering.

Generate synthetic CRM data

Show code

Build the preprocessing pipeline

Show code 
crm_num = ["annual_spend", "tenure_months", "support_tickets", "engagement_score"]
crm_cat = ["region", "plan_type", "acquisition_channel", "industry"]

crm_preprocessor = ColumnTransformer(
    transformers=[
        (
            "num",
            Pipeline([
                ("imputer", SimpleImputer(strategy="median")),
                ("scaler", StandardScaler())
            ]),
            crm_num,
        ),
        (
            "cat",
            Pipeline([
                ("imputer", SimpleImputer(strategy="most_frequent")),
                ("onehot", OneHotEncoder(handle_unknown="ignore"))
            ]),
            crm_cat,
        )
    ]
)

X_crm = crm_preprocessor.fit_transform(crm[crm_num + crm_cat])

  • one segment is midmarket partner-led accounts with moderate spend and mixed support needs,
  • one segment is enterprise direct accounts with high spend, longer tenure, and higher engagement.

5. Recommended adaptation patterns

For real survey data

Replace the synthetic survey with your real table and:

  • map Likert items to numeric scores,
  • consider one-hot encoding for purely nominal variables,
  • compare KMeans with AgglomerativeClustering.

For real transaction data

Build a customer-feature table first:

  • recency,
  • purchase frequency,
  • spend,
  • average order value,
  • category breadth,
  • discount share,
  • return rate,
  • tenure.

Then:

  • log-transform highly skewed variables,
  • standardize,
  • test multiple values of k,
  • profile the segments in business language.

For real CRM data

Use a mixed-data preprocessing pipeline:

  • impute missing values,
  • scale numerics,
  • one-hot encode categoricals,
  • cluster transformed features,
  • summarize both means and top categories by segment.

7. What to watch for in production work

A segmentation that looks good in a notebook can still fail in practice if:

  • the features are built over inconsistent time windows,
  • the segments are not actionable,
  • the cluster solution is unstable over time,
  • one-hot encoded rare categories dominate the geometry,
  • outliers are left untreated,
  • stakeholders cannot name or use the resulting segments.

Good segmentation is not just a model-fitting exercise. It is a combination of:

  • sound feature engineering,
  • appropriate preprocessing,
  • sensible clustering,
  • careful profiling,
  • business interpretation.
